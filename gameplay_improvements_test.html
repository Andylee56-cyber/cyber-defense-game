游戏玩法改进

✅ 已实现以下游戏玩法改进：

  • Agent选择限制：只有选择防御Agent后才能射击
  • 威胁类型控制：一段时间内只生成一种类型的威胁
  • 射击机制优化：只要击中就能掉血，不再检查防御类型匹配
  • 下落速度控制：降低威胁下落速度，提高可玩性
  • 射击范围限制：威胁沉到指挥官射击范围以下时扣分

改进详情

1. Agent选择限制：
- 未选择Agent时无法射击
- 选择Agent后自动射击
- 控制台显示选择提示

2. 威胁类型控制：
- 每600帧切换威胁类型
- 同时最多存在2个威胁
- 威胁生成频率降低到180帧

3. 射击机制优化：
- 只要击中就能造成伤害
- 正确防御类型给予更高奖励
- 普通消灭给予基础奖励

4. 下落速度控制：
- 威胁下落速度减半
- 提高游戏可玩性
- 给玩家更多反应时间

5. 射击范围限制：
- 威胁沉到射击范围以下扣分
- 自动销毁超出范围的威胁
- 增加游戏挑战性

2. js/main.js 修改

// 修改威胁生成逻辑
threatGenerate() {
  // 每180帧生成一个威胁，降低生成频率
  if (GameGlobal.databus.frame % 180 === 0) {
    if (GameGlobal.databus.threats.length < 2) {
      const threat = GameGlobal.databus.pool.getItemByClass('threat', Threat);
      
      // 控制威胁类型，一段时间内只生成一种类型
      const threatTypes = ['phishing', 'malware', 'ddos', 'data_leak'];
      const currentThreatType = threatTypes[Math.floor(GameGlobal.databus.frame / 600) % threatTypes.length];
      
      threat.init({ type: currentThreatType });
      GameGlobal.databus.addThreat(threat);
    }
  }
}

3. js/npc/enemy.js 修改

// 修改威胁初始化
init(config = {}) {
  // 支持指定威胁类型
  if (config.type) {
    threatData = this.getThreatByType(config.type);
    this.category = config.type;
  }
  
  // 降低下落速度
  this.speed = visualConfig.speed * 0.5;
}

// 修改伤害系统
takeDamage(damage, defenseType = null) {
  // 只要击中就能造成伤害
  this.health -= damage;
  
  if (this.health <= 0) {
    // 检查防御类型匹配，给予不同奖励
    if (defenseType && this.isDefenseMatch(this.category, defenseType)) {
      // 正确防御，更高奖励
      GameGlobal.databus.addScore('correct_defense', 20);
    } else {
      // 普通消灭，基础奖励
      GameGlobal.databus.addScore('threat_destroy', 10);
    }
  }
}
